# Contribuer — InnovQube

Merci de contribuer à la documentation stratégique d'InnovQube.

## Workflow recommandé
1. Fork du dépôt.
2. Créer une branche descriptive : `feature/mon-ajout` ou `fix/erreur-doc`.
3. Faire des commits atomiques et lisibles (Conventional Commits recommandés).
4. Ouvrir une Pull Request en utilisant le template fourni.

## Normes rédactionnelles
- Langue : français.
- Format : Markdown (`.md`).
- Titres : utilisation cohérente des niveaux `#`, `##`, `###`.
- Références : citer les sources (rapport, Statista, Gartner, INSEE, IEA) lorsque nécessaire.

## Validation
- Relecture par l'équipe produit.
- Vérification que les faits/propositions correspondent au rapport d'analyse stratégique.
